# Документация для четвертой лабораторной работы по ООАП

# Тема лабораторной: Архитектурные паттерны

---

Предметная область: Библиотека

1. Проблема: При работе с базой данных в системе библиотеки возникала ситуация, когда одни и те же объекты — например, данные о читателе или книге — запрашивались многократно в рамках одной операции.
   Каждый такой запрос выполнялся напрямую к базе данных, даже если эта информация уже была получена ранее. Это приводило к избыточной нагрузке на базу данных,  увеличению времени отклика и неэффективному использованию ресурсов.
2. Решение: Для устранения этой проблемы был применён паттерн проектирования Identity Map (Карта идентичности). Его суть заключается в том, что в памяти приложения создаётся специальная коллекция, в которой хранятся уже загруженные из базы объекты.
При поступлении нового запроса система сначала проверяет эту коллекцию. Если нужный объект уже там есть — он возвращается из памяти, без обращения к базе данных. Если объекта нет — выполняется запрос к базе, после чего результат сохраняется в коллекции для будущего использования.
