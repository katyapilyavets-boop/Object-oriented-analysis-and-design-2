# Документация для третьей лабораторной работы по ООАП

## Тема лабораторной: Поведенческие паттерны


Предметная область: Текстовый редактор

1. Проблема:  В обычном GUI-приложении сложно реализовать кнопку «Отменить» (Undo), потому что действия разбросаны по разным функциям-обработчикам.
Программа не «помнит», что именно она сделала последней, и не знает, как это откатить назад.

2. Решение: Паттерн Command превращает каждое действие (клик, ввод текста) в отдельный объект. Этот объект хранит в себе всё необходимое, чтобы выполнить действие и, главное, отменить его.
Мы складываем эти объекты в список (историю), и кнопка Undo просто берет последний объект и говорит ему: «Отмени себя».

+ Command (Абстрактная команда) — Интерфейс/Шаблон
  За что отвечает: Задает стандарт. Говорит: «Любая команда обязана иметь методы execute() (сделать) и undo() (отменить)».

+ Receiver (Получатель) — Исполнитель
  За что отвечает: Знает, как физически менять данные. Он работает с текстовым полем (tk.Text). Он не знает про кнопки или историю, он просто умеет вставлять и удалять текст.

+ Concrete Command (Конкретная команда) — Инструкция
  За что отвечает: Связывает действие с исполнителем.
  Перед выполнением запоминает состояние («было»).
  Выполняет действие через Receiver.
  При отмене восстанавливает состояние «было».

+ Invoker (Инициатор/Менеджер) — Диспетчер
  За что отвечает: Хранит историю действий (стек). Когда приходит новая команда, он её выполняет и кладет в архив. Когда жмут Undo, он достает последнюю команду из архива и вызывает у неё undo().
  В коде: Класс CommandManager.

+ Client (Клиент) — Сборщик
  За что отвечает: Создает все объекты (Receiver, Invoker), настраивает GUI и решает, какую команду создать при нажатии на кнопку.

4. Вывод

+ Приложение использует паттерн Command для реализации механизма Undo.
+ Каждое изменение текста (добавление строки, очистка, ручной ввод) упаковывается в объект-команду.
+ Команда передается в CommandManager (Invoker), который сохраняет её в списке history.
+ При нажатии Undo менеджер извлекает последнюю команду и вызывает её метод undo(), который возвращает текстовое поле в предыдущее состояние.
+ Это позволяет отменять как программные действия (кнопки), так и ручной ввод пользователя.

  В коде: Класс App.
